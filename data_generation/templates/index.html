GitHub Repositories

Instructions: Add GitHub repository URLs that contain the code you want to use for fine-tuning dataset generation. The system will clone these repositories, analyze the code, and generate Q&A pairs, completion tasks, debugging tasks, and more.

Research Papers Directory

Instructions: Provide the full path to the directory containing your lab's research papers (PDF files). The system will process these papers to generate Q&A pairs and combine them with the code dataset.
Example: /home/user/lab_papers or C:\Users\username\Documents\research_papers

Processing Pipeline

Step 1: Code Analysis
Clone repositories and analyze code files to generate diverse training data
Step 2: Paper Processing
Extract text from research papers and generate Q&A pairs
Step 3: Dataset Creation
Combine code and paper data into training and validation datasets
Generated Output Files

  • combined_dataset_val.json - Validation dataset for model evaluation
  • code_combined_train_dataset.json - Intermediate code-only training data
  • code_combined_val_dataset.json - Intermediate code-only validation data
